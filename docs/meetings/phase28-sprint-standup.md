**问题描述**: E2E 测试 ApplicationContext 加载问题持续 10+ Sprint 未解决。

**历史记录**:
- Sprint #197: MIN-3844 建立 E2E 测试基础设施 → 未完成
- Sprint #181: MIN-3753 Controller 层测试覆盖率 → 未完成
- Sprint #182: MIN-3759 单元测试覆盖率 50%+ → 进行中
- Sprint #202: MIN-3865 E2E 测试基础设施修复 → 进行中
- Sprint #203: MIN-3891 E2E 测试基础设施最终修复 → 已完成，但 Sprint #204 又出现

**根因分析**:
1. 问题复杂度被低估
2. 每次修复后 CI 环境变化导致回归
3. 缺少独立的 E2E 测试环境

**建议**:
- 单独创建 E2E 测试修复 Issue，专注解决 ApplicationContext 问题
- 考虑使用 testcontainers 进行数据库 mock
