# 微信小程序类目选择说明

## 一、类目选择原则

微信小程序的类目选择应遵循以下原则：
1. **真实性**：类目必须与实际功能匹配
2. **准确性**：选择最贴合业务的主类目和辅类目
3. **合规性**：确保有所需的资质证明

## 二、账户类型与推荐类目

### 个人开发者账户

**推荐类目**：工具 > 信息查询

| 要求 | 说明 |
|------|------|
| 资质要求 | 无需资质 |
| 功能限制 | 部分支付和营销功能受限 |
| 适用场景 | 商品信息展示、查询工具 |

### 企业开发者账户

**推荐类目**：电商平台 > 实物电商

| 要求 | 说明 |
|------|------|
| 资质要求 | 营业执照 |
| 功能限制 | 较少 |
| 适用场景 | 实物商品交易 |

## 三、当前账户类型

根据 Sprint #80 的审核材料准备（MIN-948），当前账户为**个人开发者账户**。

因此，推荐选择：
- **主类目**：工具
- **辅类目**：信息查询

## 四、审核话术参考

### 功能描述示例

```
本小程序是一款商品信息查询和购买工具，用户可以：
1. 浏览商品列表和详情
2. 将商品加入购物车
3. 创建订单并使用微信支付完成购买
4. 查看订单状态和管理订单

主要功能为商品信息展示和在线购买，属于工具类应用。
```

### 审核注意事项

1. **避免**：将小程序描述为「电商平台」或「在线商城」
2. **强调**：工具属性、信息查询功能
3. **说明**：微信支付用于购买小程序内展示的商品

## 五、备选方案

如审核被拒，可能的原因和建议：

| 原因 | 建议 |
|------|------|
| 类目与功能不符 | 调整为更贴合的类目 |
| 资质缺失 | 企业账户需补充营业执照 |
| 功能描述不清 | 补充更详细的功能说明 |
