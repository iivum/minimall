# 测试任务检查清单 / Test Task Checklist

## 概述

本文档为测试相关任务提供标准化的检查清单，确保所有测试交付物经过强制验证，防止虚假交付。

---

## 测试任务执行流程

### 1. 任务接收

- [ ] 阅读 issue 描述，确认任务范围
- [ ] 确认测试类型（单元测试/集成测试/E2E）
- [ ] 确认目标覆盖率
- [ ] 确认截止日期

### 2. 任务规划

- [ ] 分析现有测试覆盖情况
- [ ] 识别需要补充测试的模块
- [ ] 列出所有需要创建的测试文件
- [ ] 估算工作量

### 3. 测试实现

- [ ] 创建测试文件
- [ ] 编写测试用例
- [ ] 确保测试通过 (mvn test)
- [ ] 记录覆盖率数据

### 4. 交付物验证 (强制执行)

在提交任何交付物之前，**必须**执行以下验证：

#### 4.1 文件存在性验证

```bash
./scripts/verify-deliverables.sh \
  src/test/java/com/minimall/controller/XXXControllerTest.java \
  src/test/java/com/minimall/service/XXXServiceTest.java \
  target/surefire-reports/TEST-*.xml
```

**验证标准:**
- 退出码必须为 0
- 所有文件状态显示 ✅

#### 4.2 覆盖率验证

```bash
# 运行测试并生成覆盖率报告
mvn clean test jacoco:report

# 检查覆盖率
open target/site/jacoco/index.html
```

**验证标准:**
- 整体覆盖率 ≥ 目标值
- 核心模块覆盖率 ≥ 70%

#### 4.3 测试通过验证

```bash
mvn test -B
```

**验证标准:**
- 所有测试通过
- 无编译错误
- 无测试失败

### 5. 验证报告生成

- [ ] 填写验证报告 (见 `deliverables-verification-report-template.md`)
- [ ] 确认所有勾选项目已完成
- [ ] 提交验证报告到 issue

### 6. 交付标准

- [ ] 所有测试文件已创建并通过验证
- [ ] 覆盖率报告生成
- [ ] JaCoCo 检查通过
- [ ] 验证报告已提交

## 测试类型验证标准

### 单元测试

| 检查项 | 标准 |
|--------|------|
| 文件位置 | `src/test/java/com/minimall/` |
| 命名规范 | `*Test.java` |
| 测试通过率 | 100% |
| 覆盖率 | 根据任务目标 |

### 集成测试

| 检查项 | 标准 |
|--------|------|
| 文件位置 | `src/test/java/com/minimall/integration/` |
| 命名规范 | `*IntegrationTest.java` |
| 测试通过率 | 100% |
| 数据库状态 | 隔离/清理 |

### E2E 测试

| 检查项 | 标准 |
|--------|------|
| 文件位置 | `src/test/e2e/` |
| 命名规范 | `*.e2e.spec.ts` |
| 测试通过率 | 100% |
| 环境配置 | 真实环境或模拟 |

## 常见问题处理

### 验证失败

如果 `verify-deliverables.sh` 返回非零退出码：

1. 检查缺失的文件
2. 确认文件路径正确
3. 修复后重新验证

### 覆盖率未达标

1. 分析 JaCoCo 报告中的未覆盖行
2. 识别需要补充测试的代码路径
3. 补充测试用例
4. 重新运行覆盖率检查

### 测试失败

1. 查看 `target/surefire-reports/TEST-*.xml` 获取失败详情
2. 修复测试代码或被测代码
3. 重新运行测试
4. 确认所有测试通过后再交付

## 惩戒机制

根据 `team-driven-verification.md` 中的规定：

| 次数 | 惩戒措施 |
|------|----------|
| 第1次 | 警告 - 在 issue 中记录违规 |
| 第2次 | 暂停 - 该 Agent 暂停接新任务 24 小时 |
| 第3次 | 移除 - 该 Agent 从项目移除 |
