# 代码合并验证清单

## 目的
确保所有合并到 main 分支的代码符合质量标准，防止技术债务积累。

## 合并前检查清单

### 1. 代码质量
- [ ] Checkstyle 检查通过 (`mvn checkstyle:check`)
- [ ] 没有新增 Checkstyle violations
- [ ] 单元测试通过 (`mvn test`)
- [ ] 集成测试通过（如有）

### 2. 安全检查
- [ ] 没有硬编码的密钥或凭证
- [ ] 用户输入已正确验证
- [ ] SQL 注入防护到位
- [ ] 认证/授权逻辑正确

### 3. 性能检查
- [ ] 数据库查询有适当索引
- [ ] 没有 N+1 查询问题
- [ ] 适当使用缓存
- [ ] 大文件操作有分页处理

### 4. 代码规范
- [ ] 遵循项目的命名规范
- [ ] 函数不超过 50 行
- [ ] 文件不超过 800 行
- [ ] 没有深度嵌套（>4层）

### 5. Git 规范
- [ ] 提交信息遵循规范格式
- [ ] 分支命名清晰
- [ ] 没有提交敏感信息
- [ ] 关联相关 issue

## 审查要求

- 所有 PR 必须经过至少 1 人审查
- 涉及安全/支付/认证的代码需要 2 人审查
- 高复杂度变更需要架构师审核

## 回滚计划

如发现问题需要回滚：
1. 确认最后正常工作的 commit
2. 执行 `git revert <commit-hash>`
3. 推送回滚 commit
4. 通知相关人员

## 责任归属

| 角色 | 职责 |
|------|------|
| PR 作者 | 确保所有检查通过 |
| Reviewer | 验证代码质量和安全 |
| Architect | 审批架构相关变更 |
