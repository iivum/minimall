# ADR-0001: 技术栈选型决策

## 状态
已通过

## 背景
在项目初期需要确定技术栈，以支持 MVP 阶段的快速开发和迭代。

## 决策
| 组件 | 技术选型 | 原因 |
|------|----------|------|
| 后端框架 | Spring Boot 3.2.5 (Java 17) | 生态成熟，企业级稳定，适合电商业务 |
| 数据库 | PostgreSQL + H2 | PostgreSQL 用于生产，H2 用于开发 |
| 安全 | Spring Security + JWT | 无状态认证，适合小程序场景 |
| API 文档 | SpringDoc OpenAPI 3 | 自动生成 Swagger 文档，减少维护成本 |
| 微信支付 | WeChat Pay API v3 | 官方推荐版本，功能完整 |
| 构建工具 | Maven | Java 标准构建工具 |
| 容器化 | Docker Compose | 便于本地开发和服务部署 |

## 后果
- **正面**: 开发效率高，文档自动生成
- **正面**: 社区活跃，技术支持完善
- **需关注**: Java 17 升级需要迁移成本
